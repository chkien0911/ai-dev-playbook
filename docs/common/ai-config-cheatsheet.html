CLAUDE.md  ·  ~/.claude/CLAUDE.md
CLAUDE.md
auto ~70% compliance
LoadsEvery session start, resume, compaction. Walks up directory tree (nested files supported).
PurposeProject-wide persistent memory — tech stack, conventions, build commands.
Limit~150 meaningful lines max. Beyond that, Claude ignores the middle.
  • Tech stack, banned libraries, architectural decisions
  • Build & test commands
  • Project-specific idioms Claude wouldn't know
  • Safety-critical rules (use Hooks)
  • Rules for a specific directory (use .claude/rules/ with paths)
.github/copilot-instructions.md
copilot-instructions.md
auto probabilistic
LoadsEvery Chat and Agent Mode request. No directory walk — single file only.
PurposeRepo-wide baseline for all Copilot requests across all IDEs and GitHub.com.
NotePR Code Review path is slightly different — if rules don't appear, reassign reviewer to reload.
  • Team-wide coding standards, naming conventions
  • Banned libraries, preferred frameworks
  • Build & test commands for coding agent
  • File-type specific rules (use .instructions.md with applyTo)
  • Safety enforcement (use Hooks)
≈ CLAUDE.md
.claude/rules/*.md
Rules / .claude/rules/
auto probabilistic
No pathsLoaded every session — same priority as CLAUDE.md. Good for organizing global rules by topic.
With pathsLoaded on demand when Claude reads a file matching the glob. ⚠ Does not trigger on write without prior read.
---
paths:
- "src/**/Consumers/**"
---
  • MassTransit consumer rules → paths: ["src/**/Consumers/**"]
  • xUnit test conventions → paths: ["**/*.test.cs"]
  • API design rules (global, no paths) → always loaded
.github/instructions/*.instructions.md
Path-Specific Instructions
auto probabilistic
LoadsWhen Copilot processes request in context of files matching applyTo glob. Merged with global instructions.
⚠ RequiredMust set applyTo for auto-loading. Files without it are ignored automatically.
excludeAgent"code-review" or "cloud-agent" — prevent one agent from reading this file.
---
applyTo: "**/*.test.ts"
excludeAgent: "code-review"
---
≈ .claude/rules/ with paths
AGENTS.md — Cross-Tool Standard auto
WhatOpen standard (Linux Foundation, Dec 2025). Portable across Copilot, Claude Code, Codex CLI, Cursor, Gemini CLI.
Claude CodeReads AGENTS.md from repo root alongside CLAUDE.md.
CopilotReads alongside copilot-instructions.md for Chat + Coding Agent. Not supported by Code Review.
  • Team uses both Copilot and Claude Code → single source of truth
  • Multi-tool org wanting one shared file
  • Copilot Code Review — keep copilot-instructions.md too
  • Don't duplicate same rules into both files

.claude/skills/*/SKILL.md  ·  .claude/commands/*.md
Skills & Commands
semi-auto manual /name
SkillsDirectory + SKILL.md. Startup: Claude reads only name + description (~100 tokens). Full body loads only when task matches. Supports supporting files + scripts (scripts execute; only stdout enters context).
CommandsSingle .md file. Trigger only via /name. Simpler — no progressive disclosure. Use when no supporting files needed.
Key frontmatter (SKILL.md)
descriptionDiscovery trigger. Third person. Include what + when.
disable-model-invocation: trueManual only (user types /name). Claude won't auto-invoke.
user-invocable: falseClaude-only background knowledge. Never a slash command.
allowed-toolsRestrict tools (e.g. Read Grep = read-only).
context: forkRun in isolated subagent context.
.github/skills/*/SKILL.md  ·  .github/prompts/*.prompt.md
Skills & Prompt Files
semi-auto manual attach
SkillsSame SKILL.md format as Claude Code. Portable — same file works in both tools. Install path differs: .github/skills/ (workspace) or ~/.copilot/skills/ (personal).
Prompt filesDeveloper manually attaches to a Chat session. Never auto-triggers. Support variable placeholders + mode frontmatter to scope to an agent.
  • Skills: incident triage, release checklist, domain workflows
  • Prompt files: PR review, doc generation (attach when needed)
  • Rules always active → use instructions files instead
  • Safety enforcement → use Hooks
Skills ≈ Claude Code Skills (identical format)
Prompt files ≈ Commands
.claude/agents/*.md
Agents (Subagents)
auto-delegate explicit
ContextFully isolated context window. Main session stays clean. Agent returns a summary.
DelegationMain Claude auto-spawns when task matches description, or developer says "use a subagent to…"
Key frontmatter
descriptionCritical — controls auto-delegation matching.
toolsAllow list (e.g. Read, Grep, Glob = read-only).
modelopus for high-stakes, haiku for fast/cheap.
skillsPre-load skill content into agent at startup.
  • Audit 50 files without filling main context
  • Security reviewer: read-only tools, no file edits
  • Parallel builder + validator pattern
.github/agents/*.agent.md
Custom Agents
manual select
ContextReplaces default Copilot persona in the session. Developer selects from Chat panel dropdown or assigns to GitHub issue.
HandoffsUnique Copilot feature — buttons after response that transition to the next agent with pre-filled context. Enables guided Plan → Implement → Review workflows.
handoffs:
- label: "Start Implementation"
agent: implementation
send: false
